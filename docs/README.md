# Documentacion HODOM

## Organizacion

- `dashboards/`: guias operativas de las apps Streamlit activas.
- `models/`: modelos conceptuales y estructuras de datos estabilizadas.
- `sessions/`: bitacoras historicas de trabajo; pueden mencionar rutas o nombres previos a la normalizacion actual.
- `specs/`: especificaciones funcionales y normativas relevantes.
- `superpowers/`: material de trabajo auxiliar generado durante sesiones de implementacion.
- `audits/`: auditorias de repositorio e historia de cambios.

## Regla practica

La documentacion normativa y operativa vigente debe referirse al layout actual del repositorio. Las referencias antiguas quedan acotadas a `sessions/` y otros documentos explicitamente historicos.
